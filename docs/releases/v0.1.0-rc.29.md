# Ranch Hand v0.1.0-rc.29

**Classification: Public Preview.** Unsigned, not production-supported, and
not generally available.

## Persistent WSL service hosting

RC29 fixes local WSL deployments that appeared healthy immediately after
installation but disappeared when Windows closed the last WSL command. WSL
does not treat systemd services such as Docker as a reason to keep a
distribution running, so the idle shutdown stopped Docker, sent SIGTERM to the
RepoWrangler container, and removed Windows loopback port 8080 during
onboarding.

Ranch Hand now makes persistent WSL hosting an explicit prerequisite. It
preserves unrelated `%USERPROFILE%\.wslconfig` settings while applying:

```ini
[general]
instanceIdleTimeout=-1

[wsl2]
vmIdleTimeout=-1
```

Prerequisite setup restarts WSL once so the settings take effect, then verifies
Docker Engine and Compose. Apply refuses to create a WSL deployment while the
persistence settings are absent. RepoWrangler therefore remains available on
Windows loopback without an open WSL terminal.

## Exact-candidate verification

- Reproduced WSL shutdown and container SIGTERM on two Windows machines.
- Confirmed the VM stopped the Ubuntu distribution after the finite idle
  window even though Docker was enabled and running under systemd.
- Applied the RC29 persistence contract on Windows 11, WSL 2.7.8, and Ubuntu
  26.04 with RepoWrangler v1.0.17.
- After the previous shutdown window elapsed with no terminal or WSL command,
  Ubuntu remained running and `/health/ready` returned HTTP 200.
- GitHub administrator identity save and durable provider verification both
  returned HTTP 200 after the idle window.
- Go tests and vet, web typecheck/build, public product-artifact smoke, Windows
  validation, Linux validation, and CodeQL passed.

# Ranch Hand v0.1.0-rc.11 — Public Preview

Released 2026-07-18. This Preview resolves the repeated WSL directory-collision
dead end after a failed operation's lifecycle lock has already been removed.

## Fixed

- A WSL directory collision with no active Ranch Hand journal or installation
  record now offers **Inspect and remove Ranch Hand remnants** beside preflight.
- Cleanup revalidates the exact plan, release artifact, lifecycle inventory, and
  target collision immediately before changing the target.
- Ranch Hand removes only an exact matching ownership-marker deployment, the
  known legacy empty-marker pattern, or a completely empty dedicated directory.
- Unknown content, symlinks, changed files, invalid or mismatched markers, and
  unowned Docker resources remain protected.
- Successful cleanup automatically reruns live target preflight.

## Recover the blocked WSL target

Launch rc.11 and run live target preflight for the same WSL plan. Select
**Inspect and remove Ranch Hand remnants** beneath the directory collision.
When the refreshed preflight reports **Target is ready**, continue with
**Install in WSL**. No WSL terminal commands or alternate project name are
required.

This supersedes rc.10 for WSL testing. The executable remains unsigned and is
not GA or production-supported.

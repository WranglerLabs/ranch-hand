# Contributing

Thank you for helping improve Ranch Hand.

1. Open an issue for substantial behavior or architecture changes before implementation.
2. Fork the repository and create a focused branch.
3. Build the web UI, run `go test ./...`, `go vet ./...`, and compile the Windows executable.
4. Use a Conventional Commit and open a pull request describing behavior, security impact, and verification.

Contributions must preserve the secret-free plan contract, immutable artifact verification, loopback-only control API, and optional nature of Ranch Hand. By contributing, you agree that your contribution is licensed under Apache-2.0.

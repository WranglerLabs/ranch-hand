# Ranch Hand v0.1.0-rc.10 — Public Preview

Released 2026-07-17. This Preview corrects Compose environment interpolation
for WSL and remote Linux installs and makes affected older installs recoverable.

## Fixed

- SQLite evaluation installs supply a process-only interpolation sentinel for
  the disabled PostgreSQL profile; no database password is created or persisted.
- Compose receives the owned `.env` explicitly even though Ranch Hand invokes
  the verified Compose files by absolute path.
- **Remove failed installation and release lock** can remove an otherwise valid
  rc.9-or-earlier failed install whose `.env` lacks `POSTGRES_PASSWORD`.
- Cleanup failures preserve bounded Compose diagnostics.

## Recovery from rc.9

Start rc.10, open **Interrupted lifecycle work**, and select **Remove failed
installation and release lock** for the affected WSL deployment. Ranch Hand
still verifies the marker, deployment-file hashes, and Docker ownership labels
before removing anything. After cleanup completes, run preflight and install
again.

This supersedes rc.9 for all WSL and remote Linux testing. The executable remains
unsigned and is not GA or production-supported.

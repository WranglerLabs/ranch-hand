# Ranch Hand v0.1.0-rc.3 — Public Preview

Released 2026-07-17. This Preview corrects the local deployment-target model.

## Fixed

- Added **Local Docker Compose — WSL** as a first-class target.
- Kept the existing Windows Docker API implementation under the accurate
  **Local Docker Desktop** choice.
- Retained **Remote Linux Docker Compose**, **Cloudflare**, and **Azure Container
  Apps** as distinct targets.
- WSL distributions are detected automatically; Docker Desktop and SSH are not
  used for the local WSL path.
- The WSL Compose project defaults to `repo-wrangler`; its Docker-managed volume
  is `repo-wrangler-data`, and the Windows endpoint is
  `http://127.0.0.1:8080`. Users do not enter a volume path, WSL IP, or listen
  address.
- Preflight distinguishes an installed Docker CLI from a running WSL Docker
  Engine and provides a specific blocked check when the service is unavailable.

## Preview boundary

The WSL target currently supports a new evaluation install. Its complete
backup/update/restore/rollback/repair/uninstall lifecycle remains a GA gate. The
executable remains unsigned and is not GA or production-supported.

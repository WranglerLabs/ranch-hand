# Ranch Hand v0.1.0-rc.16

**Classification: Public Preview.** This release is unsigned, intended for
evaluation and feedback, and is not a production-supported or GA release.

## Simpler Remote Linux login

Remote Linux now defaults to the ordinary username-and-password workflow and
shows one password field. Operators who normally use a private key can select
that method and see only the key fields. The screen identifies the exact
`user@host:port` connection and tests it with one action.

Ranch Hand captures the server's public fingerprint before authentication.
The normal path is **Get server fingerprint** and **Trust this server key**.
Manual entry and out-of-band comparison are retained under advanced options.

After preflight succeeds, Ranch Hand retains the credential only in the running
loopback session and uses it for installation without asking again. A failed
credential remains available for correction. It is never written to the
deployment plan, lifecycle journal, logs, diagnostics, or disk.

The same one-entry rule now applies to Azure ARM and Cloudflare API tokens.
Their installation panels use the successful preflight token rather than
requesting it again. Docker Desktop requires no deployment credential and
continues directly from its native named-pipe preflight to installation.

Host-key mismatches and rejected Linux credentials now have separate errors.
